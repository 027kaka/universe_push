# 订阅服务
此服务实际为推送的基础服务，用户生成当前用户的唯一token，该服务将作为微服务发布
设备接入原则上是不需要任何参数的，但是为了以后能够迅速找到与该设备对应的channel通道，需要一个唯一的链接标识用于标识这个链接。

## 订阅协议说明

* 请求消息体为空

* 响应格式

```json
{"status":200,"token":"服务端返回的推送token，全局唯一，标识用户的唯一链接"}
```
